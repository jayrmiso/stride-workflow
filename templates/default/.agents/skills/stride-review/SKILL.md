# Stride Review

Use this skill when evaluating work for correctness, risk, and fit.

## Use when

- The user wants a review.
- The change needs a second-pass check.
- You need to identify bugs or regressions before landing.

## Do

- Focus on risks and behavior.
- Call out gaps clearly.
- Keep the review actionable.
